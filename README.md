# Lorem E-Commerce Backend

A robust backend service for the Lorem e-commerce web application, built with Go 1.25, Echo, Huma, and PostgreSQL.

## Architecture Summary
The application follows a modular, layered architecture:
- **Routers/Handlers:** Echo framework for HTTP routing with Huma for OpenAPI integration.
- **Services:** Business logic layer for operations like Emails, WebSockets, Payments.
- **Repositories:** Data access layer interacting with PostgreSQL and S3-compatible Object Storage.
- **Middlewares:** Authentication, logging, and CORS.

## Tech Stack
- **Language:** Go 1.25
- **Framework:** Echo v4
- **API Documentation:** Huma v2 (OpenAPI 3.1)
- **Database:** PostgreSQL 18
- **Payments:** Stripe
- **Storage:** AWS S3 (or compatible)

## Prerequisites
- **Go 1.25+**
- **Docker & Docker Compose**
- **Make**

## Setup Instructions

### Environment Setup
Create a `.env` file in the root directory and copy the contents from `.env.example`:
```sh
cp .env.example .env
```

### Docker Compose (Development)
The development environment includes hot-reloading using `Air` and spins up the PostgreSQL database automatically.
```sh
make dev-up
```
To stop the environment:
```sh
make dev-down
```

### Docker Compose (Production)
The production compose setup utilizes the optimized multi-stage `Dockerfile`, running the service securely as a non-root user, and configures necessary Docker health checks and restart policies.
```sh
make prod-up
```

### Bare-metal Setup
If you want to run the application directly on your machine without Docker:
1. Ensure your PostgreSQL database is running.
2. Setup the `.env` file.
3. Run `go mod download`.
4. Run `go run ./cmd/server/main.go`.

## Environment Variables Reference
Below is the list of expected environment variables (as found in `.env.example`):
- `PORT`: The port your app runs on.
- `DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_PORT`: PostgreSQL credentials.
- `JWT_SECRET`, `JWT_EXPIRE`: JWT token secret and expiration time.
- `FRONTEND_URL`: URL to your frontend client (CORS).
- `S3_URL`, `BUCKET_NAME`, `AWS_ACCESS_KEY`, `AWS_SECRET_KEY`, `AWS_REGION`: AWS S3 credentials for file uploads.
- `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_SESSION_EXPIRE`: Stripe payment API credentials.
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM`: SMTP Email service credentials.

## Available Make Targets
- `dev-up`, `dev-down`, `dev-restart`, `dev-logs`, `dev-clean`: Manage development Docker environment.
- `prod-up`, `prod-down`, `prod-restart`, `prod-logs`, `prod-clean`: Manage production Docker environment.
- `db-up`, `db-down`, `db-restart`, `db-clean`: Manage just the database environment.
- `seed-db`: Seed the database with initial data.
- `lint`: Format and vet the code.
- `test`: Run unit tests.
- `coverage-check`: Run unit tests and enforce a strict 80% coverage threshold.
- `pre-commit`: Run linting and tests prior to commit.
